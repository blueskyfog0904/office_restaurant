# 세션 복구 QA 체크리스트

1. **모바일 브라우저 백그라운드 복귀**
   - 카카오 로그인 후 사파리/크롬에서 1시간 이상 앱을 백그라운드로 둡니다.
   - 다시 포그라운드로 복귀했을 때 지역 드롭다운과 음식점 검색이 즉시 동작해야 합니다.
2. **네트워크 오프라인 → 온라인 전환**
   - 로그인 상태에서 기기를 비행기 모드로 전환한 뒤, 지역 검색을 시도해 오류 메시지를 확인합니다.
   - 네트워크를 복구하면 자동 새로고침이 실행되어 다시 데이터를 불러와야 합니다.
3. **다른 탭에서 로그아웃**
   - 동일 브라우저의 새 탭에서 로그아웃을 수행한 뒤 본 탭으로 돌아옵니다.
   - 본 탭은 세션 만료 알림 후 로그인 페이지로 이동해야 합니다.
4. **토큰 강제 만료 시**
   - 브라우저 개발자 도구에서 `localStorage`에 저장된 Supabase 토큰을 삭제합니다.
   - 다음 데이터 요청 시 “세션이 만료되었습니다” 알림이 뜨고 재로그인을 유도해야 합니다.
5. **장시간 유휴 상태 (10분 이상 입력 없음)**
   - 페이지에 아무 입력 없이 최소 10분 이상 대기합니다.
   - 자동 토큰 점검 이후에도 추가 로그인 없이 지역/검색이 정상 요청되는지 확인합니다.
